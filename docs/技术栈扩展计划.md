#### 1. RAG 智能问答助手

**业务价值**：校园社区积累大量生活/学习帖子，用 RAG 让用户可以"问社区"——输入"图书馆几点关门"，系统从历史帖子中检索相关内容，经大模型整合后生成带引用来源的回答。

**技术架构**：

```
用户提问
  ↓
Embedding 模型向量化（通义千问 text-embedding-v3）
  ↓
Milvus 向量数据库检索 Top-K 相似帖子（ANN 近似最近邻）
  ↓
构建 Prompt（System Prompt + 检索帖子内容 + 用户问题）
  ↓
调用 LLM（DeepSeek / 通义千问）生成回答
  ↓
返回答案 + 引用来源帖子链接
```

**核心实现**：

```java
@Service
public class RagService {

    private final MilvusClient milvusClient;
    private final EmbeddingModel embeddingModel;
    private final ChatClient chatClient;

    /**
     * 帖子发布/更新时，异步向量化入库（由 RabbitMQ Consumer 触发）
     */
    public void indexPost(Post post) {
        String text = post.getTitle() + "\n" + post.getContent();
        // 分块：按段落切分，每块不超过 512 tokens
        List<String> chunks = TextSplitter.splitByParagraph(text, 512);
        for (String chunk : chunks) {
            float[] vector = embeddingModel.embed(chunk);
            milvusClient.insert("post_vectors", new InsertParam.Builder()
                .withField("post_id", post.getId())
                .withField("chunk_text", chunk)
                .withField("vector", vector)
                .build());
        }
    }

    /**
     * RAG 检索 + 生成
     */
    public RagResponse ask(String question) {
        // 1. 向量化用户问题
        float[] queryVector = embeddingModel.embed(question);

        // 2. Milvus 检索 Top-5 相似帖子片段
        SearchResult result = milvusClient.search("post_vectors", queryVector, 5);
        List<String> contexts = result.getChunkTexts();
        List<Long> sourcePostIds = result.getPostIds();

        // 3. 构建 Prompt
        String prompt = buildRagPrompt(question, contexts);

        // 4. 调用 LLM 生成回答
        String answer = chatClient.call(prompt);

        return new RagResponse(answer, sourcePostIds);
    }

    private String buildRagPrompt(String question, List<String> contexts) {
        return """
            你是校园社区的智能助手。请根据以下社区帖子内容回答用户问题。
            如果帖子内容无法回答该问题，请如实说明。不要编造信息。
            
            【社区帖子内容】
            %s
            
            【用户问题】
            %s
            """.formatted(String.join("\n---\n", contexts), question);
    }
}
```

#### 2. AI 内容审核

**业务价值**：现有 TipPost 审核流程为纯人工，加入 AI 后变成"AI 初审 + 人工复审"，自动放行率可达 90%+，大幅降低审核延迟。

**技术架构**：复用现有 RabbitMQ 消息架构，新增 AI 审核 Consumer

```
用户发帖 / 评论
  ↓
发送至 RabbitMQ 审核队列（content.audit.queue）
  ↓
AI 审核 Consumer
  ├── 调用 LLM API 进行内容安全分析
  │   └── 返回结构化 JSON：{result: PASS/SUSPECT/REJECT, reason: "...", category: "..."}
  ├── PASS → 自动放行，更新帖子 status=1
  ├── SUSPECT → 标记待人工复审，MQ 通知管理员
  └── REJECT → 自动拦截，MQ 通知用户修改原因
```

**核心实现**：

```java
@Component
public class ContentAuditConsumer {

    private final ChatClient chatClient;
    private final PostMapper postMapper;
    private final MessageProducer messageProducer;

    @RabbitListener(queues = "content.audit.queue")
    public void handleAudit(Long postId, Channel channel, Message msg) {
        try {
            Post post = postMapper.selectById(postId);
            if (post == null) {
                channel.basicAck(msg.getMessageProperties().getDeliveryTag(), false);
                return;
            }

            AuditResult result = auditContent(post.getTitle(), post.getContent());

            switch (result.getVerdict()) {
                case PASS -> {
                    post.setStatus(1);
                    postMapper.updateById(post);
                }
                case SUSPECT -> {
                    post.setStatus(2); // 待人工复审
                    postMapper.updateById(post);
                    messageProducer.sendNotify(/* 通知管理员 */);
                }
                case REJECT -> {
                    post.setStatus(3); // 已拦截
                    postMapper.updateById(post);
                    messageProducer.sendNotify(/* 通知用户 */);
                }
            }
            channel.basicAck(msg.getMessageProperties().getDeliveryTag(), false);
        } catch (Exception e) {
            channel.basicNack(msg.getMessageProperties().getDeliveryTag(), false, true);
        }
    }

    private AuditResult auditContent(String title, String content) {
        String prompt = """
            你是内容安全审核员。请分析以下用户发布的内容，返回 JSON 格式审核结果。
            审核维度：色情低俗、暴力血腥、政治敏感、广告引流、人身攻击。
            
            返回格式：{"verdict":"PASS|SUSPECT|REJECT","reason":"原因","category":"违规类别"}
            
            【标题】%s
            【内容】%s
            """.formatted(title, content);

        String response = chatClient.call(prompt);
        return JSON.parseObject(response, AuditResult.class);
    }
}
```

**降级策略**：LLM API 不可用时降级为纯人工审核（设置 status=2），保障服务可用性。

**简历写法**：
> 设计 AI + 人工双层内容审核管线：发帖通过 RabbitMQ 异步投递至审核队列，Consumer 调用大模型进行内容安全分析，基于 Structured Output 解析审核结论（合规/疑似/违规），实现自动放行、人工复审、即时拦截的分级策略；LLM 不可用时降级为人工审核，保障服务可用性

**面试高频问题**：

#### 3. 语义搜索（Elasticsearch + 向量混合检索）

**业务价值**：传统关键词搜索输入"考试周怎么复习"只能匹配到"考试""复习"，语义搜索可以召回"期末备考经验""刷题技巧"等语义相关帖子。

**技术架构**：BM25 关键词检索 + 向量语义检索，RRF 融合排序

```
用户搜索 "考试周怎么复习"
  ↓
┌─────────────────────────────────────────┐
│  路径 A: Elasticsearch BM25            │ ──→ 精确关键词匹配结果
│  路径 B: Embedding → Milvus kNN       │ ──→ 语义相似结果
└─────────────────────────────────────────┘
  ↓
RRF（Reciprocal Rank Fusion）融合两路排序
  ↓
返回混合搜索结果 + 关键词高亮
```

**核心实现**：

```java
@Service
public class HybridSearchService {

    private final ElasticsearchRestTemplate esTemplate;
    private final MilvusClient milvusClient;
    private final EmbeddingModel embeddingModel;

    public List<SearchResultVo> hybridSearch(String keyword, int page, int size) {
        // 路径 A: ES BM25 关键词检索
        NativeSearchQuery esQuery = new NativeSearchQueryBuilder()
            .withQuery(QueryBuilders.multiMatchQuery(keyword, "title", "content")
                .field("title", 3.0f)  // 标题加权
                .field("content", 1.0f))
            .withHighlightFields(
                new HighlightBuilder.Field("title").preTags("<em>").postTags("</em>"),
                new HighlightBuilder.Field("content").preTags("<em>").postTags("</em>"))
            .build();
        List<RankedDoc> bm25Results = esTemplate.search(esQuery, PostDocument.class);

        // 路径 B: 向量语义检索
        float[] queryVector = embeddingModel.embed(keyword);
        List<RankedDoc> vectorResults = milvusClient.search("post_vectors", queryVector, size * 2);

        // RRF 融合排序
        return rrfMerge(bm25Results, vectorResults, page, size);
    }

    /**
     * Reciprocal Rank Fusion：score = Σ 1/(k + rank_i)，k=60
     */
    private List<SearchResultVo> rrfMerge(List<RankedDoc> listA, List<RankedDoc> listB,
                                           int page, int size) {
        Map<Long, Double> scoreMap = new HashMap<>();
        int k = 60;
        for (int i = 0; i < listA.size(); i++) {
            scoreMap.merge(listA.get(i).getPostId(), 1.0 / (k + i), Double::sum);
        }
        for (int i = 0; i < listB.size(); i++) {
            scoreMap.merge(listB.get(i).getPostId(), 1.0 / (k + i), Double::sum);
        }
        return scoreMap.entrySet().stream()
            .sorted(Map.Entry.<Long, Double>comparingByValue().reversed())
            .skip((long) page * size).limit(size)
            .map(e -> buildResultVo(e.getKey()))
            .toList();
    }
}
```

#### 4. Caffeine + Redis 多级缓存

**业务价值**：当前项目仅使用 Redis 单级缓存，热点数据（如热帖详情、公告）每次都走网络 IO 请求 Redis。加入 Caffeine 本地缓存后，L1 命中延迟 < 1ms。

**缓存架构**：

```
请求 → Caffeine L1（JVM 堆内存，< 1ms）
        ↓ MISS
      Redis L2（分布式缓存，~2ms）
        ↓ MISS
      MySQL（数据库查询，~10ms）
        ↓ 回填 L1 + L2
```

**核心实现**：

```java
@Component
public class MultiLevelCache {

    private final Cache<String, Object> caffeineCache = Caffeine.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(Duration.ofMinutes(5))  // L1 短过期
        .build();

    private final RedisUtil redisUtil;

    @SuppressWarnings("unchecked")
    public <T> T get(String key, Class<T> type, Supplier<T> dbFallback, long redisTtlMinutes) {
        // L1
        T value = (T) caffeineCache.getIfPresent(key);
        if (value != null) return value;

        // L2
        value = (T) redisUtil.get(key);
        if (value != null) {
            caffeineCache.put(key, value);
            return value;
        }

        // DB 兜底
        value = dbFallback.get();
        if (value != null) {
            redisUtil.set(key, value, redisTtlMinutes, TimeUnit.MINUTES);
            caffeineCache.put(key, value);
        }
        return value;
    }

    /**
     * 缓存失效：先删 Redis，再删本地；
     * 多实例场景通过 RabbitMQ Fanout 广播失效事件
     */
    public void evict(String key) {
        redisUtil.del(key);
        caffeineCache.invalidate(key);
        // 广播给其他实例
        rabbitTemplate.convertAndSend("cache.invalidate.exchange", "", key);
    }
}
```

**多实例缓存一致性**：通过 RabbitMQ Fanout 交换机广播缓存失效事件，所有实例收到后清除本地 Caffeine 缓存。

**依赖**：

```xml
<dependency>
    <groupId>com.github.ben-manes.caffeine</groupId>
    <artifactId>caffeine</artifactId>
</dependency>
```

**简历写法**：
> 设计 Caffeine + Redis 两级缓存架构：L1 本地缓存命中延迟 < 1ms，L2 Redis 分布式缓存兜底；通过 RabbitMQ Fanout 广播缓存失效事件，解决多实例部署下 L1 一致性问题

**预计工期**：1 天

#### 5. 布隆过滤器（Bloom Filter）

**业务价值**：恶意用户构造不存在的帖子 ID 请求，穿透缓存直接打到数据库。布隆过滤器在缓存之前拦截不存在的 ID。

**核心实现**：

```java
@Component
public class PostBloomFilter {

    private static final String BLOOM_KEY = "bloom:post:ids";

    private final RedisUtil redisUtil;

    /**
     * 应用启动时预热：加载全量帖子 ID
     */
    @PostConstruct
    public void init() {
        // Redis Bloom Filter: BF.RESERVE bloom:post:ids 0.001 100000
        List<Long> allPostIds = postMapper.selectAllIds();
        for (Long id : allPostIds) {
            redisUtil.execute("BF.ADD", BLOOM_KEY, String.valueOf(id));
        }
    }

    /**
     * 查询前先判断：不在布隆过滤器中 → 一定不存在 → 直接返回
     */
    public boolean mightExist(Long postId) {
        return redisUtil.execute("BF.EXISTS", BLOOM_KEY, String.valueOf(postId));
    }

    /**
     * 新帖发布时加入布隆过滤器
     */
    public void add(Long postId) {
        redisUtil.execute("BF.ADD", BLOOM_KEY, String.valueOf(postId));
    }
}

// 在 PostServiceImpl 中使用
public Post getPostById(Long id) {
    if (!postBloomFilter.mightExist(id)) {
        return null;  // 拦截缓存穿透
    }
    // 继续走 多级缓存 → DB 逻辑
    return multiLevelCache.get("post:" + id, Post.class,
        () -> postMapper.selectById(id), 30);
}
```

**简历写法**：
> 引入 Redis Bloom Filter 拦截缓存穿透：预热全量帖子 ID 至布隆过滤器，查询前 O(1) 判定拦截不存在的请求，误判率控制在 0.1% 以内

**预计工期**：半天

@Component
public class NotifyConsumer {

    private final SimpMessagingTemplate messagingTemplate;
    private final RedisUtil redisUtil;

    @RabbitListener(queues = "like.notify.queue")
    public void handleLikeNotify(NotifyMessage message, Channel channel, Message msg) {
        // 1. 持久化到数据库（已有逻辑）
        notificationService.saveFromMQ(message);

        // 2. 检查用户是否在线（Redis 维护在线状态）

                String.valueOf(message.getToUserId()),
                "/queue/notifications",
                NotificationVo.fromMessage(message));
        }

        channel.basicAck(msg.getMessageProperties().getDeliveryTag(), false);
    }
}

> 基于 WebSocket + STOMP 实现服务端实时消息推送，RabbitMQ 消费者处理通知后通过 SimpMessagingTemplate 即时推送至目标用户；Redis Set 维护在线状态，离线用户降级为数据库持久化

## 扩展后简历项目描述（完整版）

**Gathering Circle - 校园社交平台**

**项目简介：** 独立开发的校园社区后端平台，实现帖子发布与社交互动、AI 内容审核、RAG 智能问答、语义搜索及热帖智能排行等核心功能

**技术栈：** Spring Boot 2.7、Spring Security、MyBatis-Plus、MySQL 8.0、Redis、RabbitMQ、Elasticsearch、Spring AI、Milvus、Caffeine、WebSocket、JWT、七牛云 OSS

- 设计 **Spring Security + JWT + Redis 无状态认证方案**：HS512 签发 JWT，用户权限序列化存入 Redis，请求鉴权从 Redis 加载角色消除数据库查询；自定义 `@RateLimit` 注解基于 Redis 滑动窗口实现多维度限流，防御暴力破解

- 基于 **RAG 架构实现社区智能问答助手**：通过 Embedding 模型将帖子向量化存入 Milvus 向量数据库，用户提问时向量检索 Top-K 相似帖子注入 Prompt 上下文，调用大模型生成带引用来源的回答；帖子增量数据通过 RabbitMQ 异步向量化入库，保障索引一致性

- 设计 **AI + 人工双层内容审核管线**：发帖通过 RabbitMQ 异步投递至审核队列，Consumer 调用大模型进行内容安全分析并解析结构化审核结论，实现合规自动放行、疑似人工复审、违规即时拦截的分级策略；DFA 前缀树前置过滤减少 LLM 调用成本

- 实现 **Elasticsearch BM25 + 向量语义混合检索**：ES 负责精确关键词匹配与 IK 中文分词，Embedding 向量检索负责语义召回，通过 RRF 融合两路排序结果，兼顾精确性与语义理解

- 设计 **Caffeine + Redis 两级缓存架构** 与 **Redis Bloom Filter 防穿透机制**：L1 本地缓存命中延迟 < 1ms，布隆过滤器 O(1) 拦截不存在的 ID 请求；通过 RabbitMQ Fanout 广播缓存失效事件解决多实例 L1 一致性问题

- 构建 **RabbitMQ 多类型交换机消息拓扑**：Direct 异步邮件、Topic 分发通知、Delayed 定时发布（支持 49 天延迟）、Fanout 系统广播；全链路 Publisher Confirms + 手动 ACK 保障可靠投递；基于 WebSocket + STOMP 将通知实时推送至在线用户
